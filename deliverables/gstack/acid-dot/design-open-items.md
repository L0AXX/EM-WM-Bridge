# 酸液 DoT — 开放设计方案（待服务器侧定稿）

> 本文件列出 T4 酸液喷射器 DoT 的未决设计点。**在用户/设计侧给出服务器侧方案前，主代码不应写入酸液逻辑。**

## 0. 根因（为什么会「落空」）
当前没有为酸液喷射器定义**命中 → 施加 DoT** 的链路：武器发射后若没有命中判定事件去触发状态施加，伤害就不会落到目标上（「落空」）。必须先确定命中判定方式，再谈 DoT 参数。

## 1. 命中判定方式（服务器侧方案，必选其一）
- **方案 A — 投射物命中事件**：酸液作为 WM 投射物，监听 `ProjectileHitEvent` / 自定义射线命中实体 → 触发 DoT 施加。
  - 前提：酸液喷射器走 WM 武器路径（则 NBT 弹药扣减见需求5 / open-questions 第1节）。
- **方案 B — 区域持续效果**：酸液落地生成酸液云/地块（自定义区域），范围内实体每 tick 受伤。
  - 需定：区域形状（圆/锥）、半径、存活时长、是否与方块/液体交互。
- **方案 C — 近身喷溅范围**：以 NPC 为中心 cone/radius 内目标施加（近战式）。
  - 需定：喷射距离、锥角、是否需视线。
- ❓ 待定：酸液喷射器本质是**远程投射**还是**近身喷溅**？这决定 A/B/C 哪个成立。

## 2. DoT 参数（待方案 + 测试服反算，禁止空占位）
- 持续时长 `duration`（ticks/秒）
- 每跳间隔 `tickInterval`
- 每跳伤害 `dmgPerTick`
- 最大层数 / 刷新规则（重复命中是否叠加或刷新）
- **护甲交互**：DoT 是否受 `Bullet_Resistance` / `MAX_HEALTH` 影响？→ 取决于需求7 验证门结论（见 open-questions 第4节）。

## 3. 阵营 / 友伤
- 酸液是否误伤同阵营？是否对中立/友方生效？
- 必须对齐 `emwm_factions.yml` 关系矩阵（`HostilityMatrix.Relation`）：仅对 `HOSTILE` 目标施加，或按配置允许 AOE 友伤。

## 4. 表现层
- 状态效果：`PotionEffect`（如 `POISON`/自定义）或粒子云
- 客户端可见性：非玩家实体上 DoT 状态是否可见（避免「无反馈」误判为落空）
- 音效 / 视觉（可选）

## 5. 与 WM 武器系统的关系
- 酸液喷射器是 WM 武器还是 EM 自定义能力？
- 若走 WM 武器路径 → 需求5 的 NBT 弹匣扣减是否适用（open-questions 第1节）。
